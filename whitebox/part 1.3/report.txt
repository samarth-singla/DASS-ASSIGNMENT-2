2) Why Extensive Module-Wise White-Box Testing Was Necessary
- MoneyPoly has many state transitions (money, ownership, jail state, mortgage state, turn index) that interact with each other.
- A bug in one module (for example Bank or Board) can silently corrupt Game decisions later.
- Module-wise white-box testing ensures each internal branch and guard is validated before integration behavior is trusted.
- High-risk edge cases (negative amounts, empty collections, insufficient reserves, invalid trade amounts) are important because they are common crash or corruption sources.
